Introduction

最終更新日時: $Date:: $

S2Directoryは、ディレクトリサーバとの接続を容易にするためのコネクション管理機能と、ディレクトリサーバのエントリ・属性とオブジェクトを結びつけるO/D Mapping ( Object / Directory Mapping )、もしくは、O/L Mapping ( Object / LDAP Mapping ) のフレームワーク機能を提供します。もっと簡単に一言で言い表しますと、S2DaoのDao対象がリレーショナルデータベースからディレクトリサーバになったものです。

Status

Note: 現在開発中のため、将来的にAPI等が変わる可能性があります。

実装済み

  • SASL (Simple Authentication and Security Layer) 認証
  • ユーザモード接続
  • 新規エントリ作成機能
  • 読み出し機能
  • 更新機能
  • 削除機能
  • バイナリデータ対応
  • パスワード対応 (PLAIN, MD5, SMD5, SHA, SSHA)
  • 複数の属性値の読み書きに対応 (java.lang.String, java.util.List)

未実装

  • SSL/TLS対応

Directory Server

ディレクトリは主にIDの一元管理のために利用されているものです。Seasar.orgサーバでもユーザ・プロジェクト管理に使用されています。主なディレクトリサーバの実装は、次に挙げる通りです。

主なディレクトリサーバ一覧

製品名 Support
OpenLDAP Server YES
Apache Directory Server YES (but, SSHA and SMD5 are not supported in ApacheDS 1.0-RC1)
Red Hat Directory Server Unknown
Fedora Directory Server Unknown
Microsoft Active Directory Server Unknown
Sun Java System Directory Server Unknown
IBM Tivoli Directory Server Unknown
Novell eDirectory Server Unknown
